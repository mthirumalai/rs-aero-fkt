# Claude's Automated Push & Monitor Workflow

This document defines my standard workflow for pushing code changes with automatic Railway deployment monitoring.

## Workflow Steps

1. **Push code changes**
   ```bash
   git push
   ```

2. **Start background monitoring**
   ```bash
   ./.railway-monitor.sh
   ```

3. **If deployment fails:**
   - Pull Railway logs
   - Analyze errors (TypeScript, build issues, etc.)
   - Fix code issues
   - Commit and push fixes
   - Restart monitoring (max 3 attempts)

4. **Status updates to user:**
   - 🚨 "Railway deployment failed, diagnosing..."
   - 🔧 "Fixed [issue], pushing fix..."
   - ✅ "Deployment successful"

## Config Management

- Edit `.railway-monitor.config` to update settings
- Current settings: 15min timeout, 30s polling, 3 max retries
- Brief notifications only

## Usage

This workflow activates automatically after every `git push` I perform.
To disable: "Stop monitoring Railway deployments"
To modify: Edit `.railway-monitor.config`
